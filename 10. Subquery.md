# MySQL Subquery

## Introduction

This lesson introduces students to **MySQL Subqueries**, which are queries nested inside another query. Students will learn how to use subqueries with `SELECT`, `WHERE`, `FROM`, comparison operators, and correlated subqueries using the `classicmodels` database.

---
# Prerequisites
Students should already know:

- Basic `SELECT`
- `WHERE`
- Aggregate functions (`MAX`, `AVG`, `COUNT`)
- `GROUP BY`
- Basic joins

---
# 1. Introduction to Subqueries

## Definition

A **subquery** is a query inside another SQL query.

- Inner query → subquery
- Outer query → main query

## Syntax

```sql
SELECT column_name
FROM table_name
WHERE column_name = (
    SELECT column_name
    FROM table_name
);
```

---

# Example 1: Employees Working in USA Offices

```sql
SELECT 
    lastName,
    firstName
FROM employees
WHERE officeCode IN (
    SELECT officeCode
    FROM offices
    WHERE country = 'USA'
);
```

## Explanation

1. The subquery finds office codes in the USA.
2. The outer query finds employees working in those offices.

---

# 2. Subquery with Comparison Operators

Subqueries can return a single value.

## Example: Highest Payment

```sql
SELECT 
    customerNumber,
    checkNumber,
    amount
FROM payments
WHERE amount = (
    SELECT MAX(amount)
    FROM payments
);
```

## Explanation

- Subquery returns the maximum payment amount.
- Outer query finds the payment record matching that value.

---

# Example: Payments Greater Than Average

```sql
SELECT 
    customerNumber,
    checkNumber,
    amount
FROM payments
WHERE amount > (
    SELECT AVG(amount)
    FROM payments
);
```

## Explanation

- Subquery calculates average payment.
- Outer query finds payments larger than average.

---

# 3. Subquery with IN and NOT IN

Used when the subquery returns multiple values.

## Example: Customers Without Orders

```sql
SELECT customerName
FROM customers
WHERE customerNumber NOT IN (
    SELECT customerNumber
    FROM orders
);
```

## Explanation

- Subquery gets customers who placed orders.
- Outer query returns customers not in that list.

---

# 4. Subquery in the FROM Clause

A subquery can act like a temporary table.

## Example: Order Item Statistics

```sql
SELECT 
    MAX(items) AS max_items,
    MIN(items) AS min_items,
    FLOOR(AVG(items)) AS avg_items
FROM (
    SELECT 
        orderNumber,
        COUNT(orderNumber) AS items
    FROM orderdetails
    GROUP BY orderNumber
) AS lineitems;
```

## Explanation

- Inner query counts items per order.
- Outer query calculates statistics from those results.

---

# 5. Correlated Subqueries

A correlated subquery depends on the outer query.

## Example: Products Above Product Line Average Price

```sql
SELECT 
    productName,
    buyPrice
FROM products p1
WHERE buyPrice > (
    SELECT AVG(buyPrice)
    FROM products
    WHERE productLine = p1.productLine
);
```

## Explanation

- The subquery runs once for each row.
- It calculates the average price within the same product line.
- Products above that average are returned.
