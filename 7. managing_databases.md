# Managing Databases

## 1. Select a Database

The `USE` statement is used to select a database for working with tables and data.

## Syntax

```sql
USE database_name;
```

## Example

```sql
USE classicmodels;
```

After selecting the database, all SQL commands will run inside that database.

---

# 2. Create Databases

The `CREATE DATABASE` statement is used to create a new database.

## Basic Syntax

```sql
CREATE DATABASE database_name;
```

## Example

```sql
CREATE DATABASE school_db;
```

---

## Create Database with CHARACTER SET and COLLATE

You can define the character encoding and collation when creating a database.

- `CHARACTER SET` defines how characters are stored.
- `COLLATE` defines sorting and comparison rules.

## Syntax

```sql
CREATE DATABASE database_name
CHARACTER SET charset_name
COLLATE collation_name;
```

## Example

```sql
CREATE DATABASE company_db
CHARACTER SET utf8mb4
COLLATE utf8mb4_unicode_ci;
```

---

## Common Character Sets

| Character Set | Description |
|---|---|
| utf8mb4 | Supports full Unicode including emojis |
| latin1 | Western European characters |

---

## Common Collations

| Collation | Description |
|---|---|
| utf8mb4_general_ci | Fast and case-insensitive |
| utf8mb4_unicode_ci | Unicode-based comparison |
| utf8mb4_bin | Case-sensitive |

---

# 3. Drop Databases

The `DROP DATABASE` statement is used to permanently delete a database.

## Syntax

```sql
DROP DATABASE database_name;
```

## Example

```sql
DROP DATABASE school_db;
```

> Warning: Dropping a database removes all tables and data permanently.

# Summary

| Command | Purpose |
|---|---|
| `USE` | Select a database |
| `CREATE DATABASE` | Create a new database |
